# Cyber Smash (赛博黑客) 项目综合介绍

## 1. 项目概述

**Cyber Smash** 是一款基于 **Godot 4.x** 引擎开发的 3D 无尽跑酷游戏，其核心特色是创新的**体感交互**。玩家通过电脑摄像头，使用手势来直接控制游戏中的体素怪兽，在充满赛博朋克风格的数字世界中穿行、破坏和得分。该项目不仅是一个完整的游戏，还包含了一个用于宣传的静态网页和一套完整的项目源代码。

---

## 2. 使用技术

该项目是一个技术栈丰富的综合性应用，主要包含以下几个部分：

*   **游戏主体 (Game Development):**
    *   **游戏引擎:** Godot Engine v4.5 (或更高)。使用其强大的 3D 渲染、物理系统和节点架构。
    *   **编程语言:** GDScript，Godot 的原生脚本语言，用于实现所有游戏逻辑，如玩家控制、关卡生成、UI交互、得分与生命值管理等。

*   **体感控制器 (Motion Control):**
    *   **编程语言:** Python 3。
    *   **核心库:**
        *   **OpenCV (`cv2`):** 用于捕获和处理来自摄像头的实时视频流。
        *   **MediaPipe (`mediapipe`):** Google 的开源框架，用于实时、高精度地识别人手关键点，判断手势（如张开、握拳）和位置。
    *   **通信方式:** Python 控制器通过 **UDP 协议** 将识别出的指令 (如 "LEFT", "JUMP") 实时发送给正在运行的 Godot 游戏，端口号为 `4242`。

*   **项目前端 (Frontend):**
    *   **技术:** 标准的 HTML5 和 CSS3。
    *   **用途:** 创建了一个精美的静态宣传网页，用于介绍游戏特色、玩法、开发团队，并提供下载指南。

*   **打包与分发 (Packaging):**
    *   游戏和 Python 控制器都被分别打包成了 Windows 可执行文件 (`.exe`)。
    *   提供了一个 `Launcher.bat` 批处理脚本，用于一键启动控制器和游戏，并在游戏结束后自动关闭控制器进程，优化了用户体验。

## 3. 实现功能

*   **核心游戏功能:**
    *   **无限跑酷:** 游戏地图会通过程序化自动、无限地生成，玩家需要不断前进挑战极限。
    *   **动态难度:** 游戏速度会随着时间推移而逐渐加快，挑战玩家的反应速度。
    *   **障碍与互动:** 赛道上布满了多种障碍物，如需要跳跃的断桥、需要通过特定手势（握拳）破坏的墙体等。
    *   **得分与收集:** 玩家可以收集金币来增加分数，也可以收集特定道具（如爱心）来恢复生命值。
    *   **特殊能力:** 游戏包含特殊道具，拾取后可触发限时能力，如吸收金币的“磁铁”和免疫伤害的“无敌护盾”。
    *   **生命与结束:** 玩家拥有有限的生命值，与障碍物碰撞会减少生命值，归零时游戏结束并显示得分。
    *   **完整的 UI:** 包含主菜单、游戏中的 HUD（显示生命、速度、得分）以及游戏结束界面。

*   **手势控制功能:**
    *   **左右移动:** 通过手掌在摄像头画面中的水平位置来控制角色向左、中、右三个赛道移动。
    *   **跳跃:** 手掌快速向上移动被识别为“跳跃”指令。
    *   **攻击/破坏:** 手掌由张开状态变为“握拳”状态被识别为“攻击”指令，用于摧毁特定障碍。
    *   **可视化窗口:** 控制器会显示一个小的摄像头预览窗口，并能实时标注识别出的手部关键点，方便用户校准自己的手势。

## 4. 目录结构解析

项目的根目录结构清晰，主要分为三个部分：

*   `CyberSmash_V1.0/`
    *   **用途:** 存放**已打包完成、可直接运行**的游戏版本。
    *   `Launcher.bat`: 推荐使用的启动器，能一键启动并管理游戏和控制器。
    *   `CyberSmash/`: 包含 Godot 游戏主程序 (`.exe`) 和资源包 (`.pck`)。
    *   `hand_controller/`: 包含已编译为可执行文件的 Python 手势控制器。

*   `前端/`
    *   **用途:** 存放用于**项目展示的静态网页**的所有源文件（HTML, CSS, 图片等）。

*   `项目源代码/`
    *   **用途:** 存放**完整的、可供二次开发的 Godot 和 Python 项目源文件**。
    *   `project.godot`: Godot 项目的配置文件，双击可用 Godot 编辑器打开。
    *   `Assets/`: 存放所有美术与音频资源，如 3D 模型、贴图、音效、音乐。
    *   `Scenes/`: 存放所有 Godot 场景文件 (`.tscn`)，包括游戏关卡、玩家、敌人、UI 界面等。
    *   `Scripts/`: 存放所有 GDScript 脚本 (`.gd`)，是游戏逻辑的核心。
    *   `Python_Controller/`: 存放手势控制器的 Python 源代码 (`.py`)。
